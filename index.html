つなぐ
ポ ー ト フ ォ リ オ

世の中には、離れているように見えて、
実は一本の線でつながっているものが多くあります。

私の名前「廉潔」は、中国語の「連接」と同じ響きを持ち、
日本語では「連結」とも読めます。

私にとってその線が「つなぐ」です。

現実の仕組みを、仮想の体験へ。仮想の物語を、誰かの記憶へ。
そして現実×仮想の境界で、人の意図を読み取り、
より自然なインタラクションへ。

技術で、「体験のつながり」を設計してきました。

多機能スマート窓

現実世界の環境変化を検知し、窓の動作へとつなぐIoTシステム

開発期間　2ヶ月
人数　5人（チーム制作）
担当　チームリーダー・電子技術・制御システム
技術　C / マイコン

環境変化（雨・ガス等）に応じて窓を安全に自動制御するIoTシステム。光・雨・ガス・人感の4種センサーからの入力を閾値判定と状態遷移で処理し、ステッピングモータと電磁石で窓を駆動。人体検知時の安全停止設計や、ESP32によるWi-Fi遠隔操作への拡張も可能。

『氷嬉図』

仮想世界で「文化と体験」をつなぐストーリーアクション

開発期間　1ヶ月
人数　2人（未経験から制作開始）
担当　企画・シナリオ・レベル設計・Unity実装
技術　C# / Unity2018.4.32f1

氷雪文化の原点「氷嬉大典」を追うため、主人公は時空シミュレーションで「清王朝」へ。遊びながら「歴史の記憶」に触れる2D横スクロールゲーム。全5ステージで操作体験を段階的に変化させ、コレクション要素で史実エピソードを解放する設計。絵そのもののように、失われた「背後の記憶」を受け取る体験へ。

中国大学生コンピューター設計大会「二等賞」受賞 全5ステージ＋ストーリー演出＋コレクション要素（約25分）

夢売店

日常の秩序と夢の狂気をつなぐタイムマネジメントゲーム

開発期間　7日間
人数　1人（個人制作）
イベント　1週間ゲームジャム（お題：「うら」）
技術　C# / Unity6000.3.9f1

コンビニバイトでの実体験をもとに、秩序だった日常業務と制御不能な夢の混乱をつないだゲームジャム作品。準備→通常営業→裏営業（夢境化）の3段階構成で、前半の整理行動が後半の生存率へ接続する設計。プレイヤーからは「裏営業から一気にゲームの面白さが押し寄せてきました！」との声も。

修士研究

現実×仮想のインタラクションをつなぐ — VR操作における短期動作予測に基づく意図推定と適応的操作支援

所属　筑波大学大学院
技術　C# / Unity / Python / PyTorch

VR空間での空中操作特有の手ブレやUI誤選択に対し、GRUモデルによる短期動作予測で意図を先読みし、操作アシストへ接続する研究。手部（コントローラ）と頭部（HMD）の位置・回転データから、100ms先の動作方向と意図を同時推定。推定結果をUnity上のハイライトやスナップ補助へリアルタイム接続する構成。

ExGRUモデル — マルチモーダル入力 意図＋対象を同時推定 リアルタイム操作支援

夢売店VR（初期試作・構想）

修士研究と『夢売店』を接続する今後の構想

技術　C# / Unity / Python
状態　初期試作・体験方向の検討段階

『夢売店』の混乱した接客体験を3DのVR空間へ拡張する構想。飛散・浮遊する商品に対し、空間内での把持・選別を行うゲーム体験。2D版よりも強い空間的混乱と没入感を実現し、修士研究で扱うGRUモデルによる把持予測・意図推定の適用先として検討中。

つなぐ、という言葉は
私にとって一本の線です。

現実と仮想のあいだ。
仕組みと体験のあいだ。

そして、失われたものの背後に残る記憶のあいだ。

私はその「あいだ」に線を引き、
人の心が動く体験をつくりたい。

未来へ向けて、つながりレンケツはまだ続きます。
